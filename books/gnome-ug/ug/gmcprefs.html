Настройка диспетчера файлов.

Вы можете изменить по своему вкусу многие параметры диспетчера файлов GNOME.

Настройка параметров GNOME осуществляется с помощью диалога GMC Preferences, который вызывается командой Настройка в меню Правка.

Диалог GMC Preferences состоит из следующих разделов: Файлы, Подтверждение, ВФС, Caching и Специальный.

Файлы

Рисунок 6-7. Раздел "Файлы"

  • В разделе Файлы Вы можете изменить настройку показа файлов в главном окне диспетчера файлов.

    • Показывать резервные файлы - в этом режиме диспетчер файлов будет показывать резервные копии файлов (backup files) на вашей системе.

    • Показывать скрытые файлы - в этом режиме диспетчер файлов будет показывать все "dot files", то есть файлы и каталоги, имена которых начинаются с точки. Как правило, это конфигурационные файлы.

    • Файлы и каталоги вперемешку - по умолчанию, диспетчер файлов всегда показывает каталоги перед файлами. В этом режиме файлы и каталоги будут показаны вместе, упорядоченные в согласии с выбранным вами полем (имя, размер,...).

    • Образцы командной строки вместо регулярных выражений - эта настройка предназначена только для опытных пользователей. Если вы незнакомы с синтаксисом регулярных выражений (Regular Expressions), то вам не следует ее менять. Пользователи с опытом работы с регулярными выражениями могут отменить этот режим и использовать регулярные выражения в качестве критериев для выборки файлов и в фильтрах.

Подтверждение

Рисунок 6-8. Настройка запросов подтверждений

  • В этом разделе вы можете указать, перед выполнением каких операций диспетчеру файлов следует запрашивать подтверждение.

    • Подтверждение при удалении файла - перед тем, как уничтожить файл, диспетчер файлов выдаст окно с предупреждением и потребует подтверждения.

    • Подтверждение перед перезаписью файлов - перед тем, как переписать файл, диспетчер файлов выдаст окно с предупреждением и потребует подтверждения.

    • Подтверждение перед запуском файлов - перед тем, как запустить файл с программой, диспетчер файлов выдаст окно с предупреждением и потребует подтверждения.

    • Show progress while operations are being performed - при выполнении различных операций (копирование файлов, перемещение, уничтожение и т. п.), диспетчер файлов будет выдавать на экране полосу, которая заполняется по мере выполнения операции.

ВФС

Рисунок 6-9. Настройка ВФС

  • В этом разделе вы можете изменить различные параметры ВФС (виртуальная файловая система).

    Виртуальная файловая система предоставляет вам доступ к файлам, которые в действительности не находятся в вашей файловой системе. В частности, она дает доступ к файлам на FTP сайтах и внутри tar архивов (tar=Tape ARchive - распространенный в UNIX формат архивных файлов).

    • VFS Timeout - этот параметр определяет, через какое время бездействия соединение с ВФС прерывается. Время измеряется в секундах.

    • Anonymous FTP password - этот параметр задает пароль для соединения с анонимными FTP сайтами. Большинство таких сайтов просят ввести в качестве пароля ваш e-mail адрес.

    • Always use FTP proxy - этот режим следует выбрать, если вы соединяетесь с FTP сайтами через прокси. Если вы не знаете, что это такое, то скорее всего, к вам это не относится.

Caching

Рисунок 6-10. Настройка cache

  • В этом разделе вы можете изменить настройку cache, используемого для ускорения работы диспетчера файлов.

    • Быстрая перезагрузка каталога - в этом режиме диспетчер файлов будет использовать cache для хранения информации о каталогах, что ускоряет открывание каталога.

      ВАЖНАЯ ИНФОРМАЦИЯ: Если выбрать Быстрая перезагрузка каталога, то может случиться, что диспетчер файлов не будет показывать вновь добавленные в каталог файлы.

    • Compute totals before copying files - в этом режиме перед тем, как начать копирование группы файлов, диспетчер файлов подсчитает их количество. Это необходимо для того, чтобы вы могли следить за прогрессом операции.

    • FTP directory cache timeout - этот параметр определяет, какое время (в секундах) диспетчер файлов хранит в cache оглавление FTP сайтов, которые вы посетили.

    • Allow customization of icons in icon view - этот режим позволяет вам изменить пиктограммы для файлов в главном окне диспетчера файлов (в режиме "Пиктограммы"). Вы всегда можете изменить пиктограмму объекта на вашем рабочем столе - для этого надо щелкнуть на него правой кнопкой мыши и выбрать в появившемся меню команду Свойства. Если вы выберете режим Allow customization of icons in icon view, то то же самое будет действовать и в главном окне диспетчера файлов.

      ВАЖНАЯ ИНФОРМАЦИЯ: Использование режима Allow customization of icons in icon view может замедлить работу вашей системы.

Специальный режим

Этот раздел позволяет вам настроить специальный режим показа в диспетчере файлов GNOME.

Рисунок 6-11. Натсройка специального режима

Этот раздел содержит два списка: Possible Columns и Displayed Columns.

Список Possible Columns содержит все возможные поля с информацией про файлы, который могут быть показаны диспетчером файлов. Чтобы показывать одно из них, выделите его и нажмите кнопку Add, и это поле будет добавлено к списку Displayed Columns.

Список Displayed Columns содержит все поля, которые показываются в настоящий момент в специальном режиме. Вы можеет удалить одно из них, выделив его и нажав кнопку Remove.

Вы можете изменить порядок показа полей, перемещая их с помощью мыши в списке Displayed Columns.
