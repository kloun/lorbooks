Сохранение и загрузка файлов

Для загрузки файла используйте горячую клавишу F1, для сохранения - F2. При нажатии этих клавиш вместо активного окна появляется окно FileWindow.
  • Левая кнопка мыши: сделать файл активным/открыть поддиректорию.
  • Средняя кнопка: загрузить/сохранить файл.
  • Для загрузки или сохранения активного файла (который выводится в поле "имя файла") нажмите ENTER.
  • Для отмены и возврата к тому окну, которое было до нажатия клавиш F1 или F2, нажмите ESC.
Две верхних поля в окне FileWindow показывают каталог и имя файла. По левой кнопке мыши кнопка становится активной, ENTER вводит в нее текст.

В Blender существует следующая защита от возможных сбоев программного обеспечения и сохранения испорченных данных:
  • Создание нескольких копий (версий) файла: В UserMenu (для его открытия потяните край окна над заголовком вверху) есть кнопка "Versions" (Версии). При включенной кнопке перед сохранением нового файла программа автоматически переименовывает старый (старые).

    Пример: файл rt.blend переименовывается в rt.blend1.

  • Авто-сохранение: При включенном "Auto save" (кнопка в UserMenu) программа будет сохранять на диск временный файл каждые х минут с именем идентификатора процесса (уникальный номер). После успешного выхода из Blender (по клавише Q) этот файл переименовывается в 'quit.blend'.
У Blender консервативное управление памятью, что иногда может обескураживать.

Например, удаление Объекта не означает, что все, что с ним связано (Каркасы, Материалы и т.д.) немедленно высвобождается из памяти. После нескольких часов работы в памяти Вашего компьютера может оставаться большое количество неиспользуемых блоков данных.

Эту особенность программы можно использовать для "отмены" (undo) удаления: с помощью кнопки поиска (browse button) в заголовках окна связи можно восстановить.

Блоки данных, неиспользуемые Объектами программы, в файл НЕ сохраняются.

Память высвобождается и реструктурируется только после загрузки файла Blender.
